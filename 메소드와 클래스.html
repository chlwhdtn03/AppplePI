접근 제어 지시자

Java에서는 C와 다르게 다양한 방식으로 여러 파일을 접근하기 때문에 혼선을 예방하기 위해 접근 규칙을 정해줘야 합니다.

어디서 접근할 수 있게 할것인가

이 규칙을 접근 제어 지시자라고 합니다.

Java에서 가장 많이 쓰이는 접근 제어 지시자는
public, private 두가지 인데,
public은 초기화만 해주면 다른 클래스에서 언제나 접근이 가능하고,
private은 그 클래스 안에서만 접근이 가능합니다.

같은 클래스는 해당 getInt()메소드가 있는 있는 클래스를 말합니다.

C언어에서는 1개의 C파일에서 작업하는 경우가 많아 public을 쓰던, private를 쓰던 어차피 모두 한 파일안에 있었기 때문에 항상 접근이 가능했겠지만,
자바에서는 여러 클래스를 사용하기 때문에 public과 private를 정확하게 사용해야 합니다.

메소드

이제 콘솔창을 띄우고 메세지를 출력합니다.

먼저 프로젝트 안에 새 클래스를 만들어줍니다.
Main이라는 이름을 가진 클래스를 만들어줍시다.
클래스 어떻게 만들어요 

public class Main {
	
}

이렇게 창이 뜬다면 정상적으로 만들어진 것입니다.

C언어에서는 int main(void) { } 를 사용했지만 자바에서는

public class Main {
	public static void main(String args[]) {
		// 실행될 코드	
	}
}

public static void main(String args[]) {} 가 실행 메소드 입니다.
순수 자바를 사용할땐 main메소드가 꼭 필요하지만, 앱 개발이나, 특수한 상황에서는 필요하지 않습니다.

이제 출력 코드를 넣어줍시다.
출력 코드는 System.out.println(변수);로 C언어에서의 printf와 유사하며 자동으로 개행시켜줍니다.

System.out.println("Hello World");

Ctrl + F11을 눌러 실행해봅시다.

자료형

설명 선언예시
int 4byte 정수 int a = 5;
long 8byte 정수 long a = 21;
float 4byte 실수 float a = 3.2;
float a = 5F
double 8byte 실수 double a = 3.5
double a = 3D
String 문자열 String str = “가나다라”
boolean True/False boolean isHappy = false;
boolean isLonely = true;

boolean은 생소할 수 있는데, 조건문에서 1, 0이 아닌, true와 false을 사용할 수 있는게 이점입니다.

메소드 만들기

public class Main {
	public static void main(String args[]) {
		System.out.println("Hello World");	
	}
}

덧셈을 해주는 메소드를 만들어줍시다.

public class Main {
	public static void main(String args[]) {
		System.out.println("Hello World");	
	}
	public int plus(int a, int b) { // 매개변수 설명하기
		// 여기에 필요한 코드??
	}
}

이렇게 plus메소드를 만들어줬는데, 실행 메소드와는 살짝 다르게 생겼습니다.

static이 없고,
메소드에도 void가 아닌 int가 들어가 있는데, 이건 반환값을 의미합니다.
void는 반환하지 않겠다는 표기이고, int와 같은 자료형이 들어가면 해당 값을 반환하겠다는 표기입니다.

반환을 할때는

return (반환값);

이런 코드를 사용합니다.
반환하는 순간 그 메소드를 빠져나오게 됩니다.

public class Main {
	public static void main(String args[]) {
		System.out.println("Hello World");	
	}
	
	public int plus(int a, int b) {
		// 여기에 필요한 코드??
	}
}

그렇다면 위 설명을 통해 작성할 수 있는 코드는 아래가 될것입니다.

public class Main {
	public static void main(String args[]) {
		System.out.println("Hello World");	
	}
	
	public int plus(int a, int b) {
		return a+b;
	}
}

하지만 여기서 실행하게 되면 아무런 변화가 없습니다.
출력문에서는 plus를 사용하지 않았기 때문인데, 출력문에 방금 만든 메소드를 넣어줍시다.

같은 클래스 안에 있는 메소드는 함수명과 매개변수만 입력하시면 됩니다.

혹시 C언어를 잊지 못하고 아래와 같이 코드를 작성했다면 에러가 발생할겁니다.

System.out.println("출력 : %d", plus(3,5)); // ^에_러^

이럴땐 아래 코드를 사용하면 정상적으로 실행이 됩니다.

System.out.printf("출력 : %d", plus(3,5)); // 출력 : 8

자바 정석대로 실행시키고 싶다면 아래처럼.

System.out.println("출력 : " + plus(3,5)); // 출력 : 8

주의!! 실행문에는 System.out.printf();라는 최소한의 배려가 있지만 원활한 개발을 위해서는 연산자 '+'에 익숙해 지는걸 권장합니다.

연산자 ‘+’

System.out.println(5+10+5); // 20
System.out.println("3"+"2") // 32
System.out.println("가나"+"abc); // 가나abc
System.out.println(1+2+"가"+5+5); // 3가55
System.out.println(60+10+"가"+(1+2)); // 70가3

+를 통해 덧셈을 하거나 문자열을 이어 붙일 수 있는데, 중간에 문자열이 나온 이후 부터는 덧셈이 불가능해지고 문자열처럼 붙어버리게 됩니다.

하지만 괄호로 연산의 우선순위를 높여주게 되면 정상적으로 덧셈이 진행됩니다.

데이터 클래스

새로운 클래스를 또 하나 만들어줍니다! 새로운 클래스의 이름은 Computer입니다.

public class Computer {

}

여기엔 실행 메소드가 들어가지 않습니다. 여기에서는 데이터를 저장하기 위한 '틀’을 만드는 작업을 하게됩니다.

전역변수라는 단어를 쓰게될텐데, 변수는 메소드 밖에도 있을 수 있습니다.
메소드 밖에 있는 변수를 전역변수라고 합니다.

주의! 그렇다고 변수가 클래스 바깥구역에 있을 순 없습니다.

모든 컴퓨터에는 IP라는 주소값을 가지고 있으니 IP값을 저장하는 문자열을 만들어줍시다. 전역변수로!

public class Computer {
	String ip;
}

지금 저 ip 변수에는 아무런 접근 제어 지시자가 없습니다! public을 넣어줍시다!
만약 이 부분에서 public이 아닌 private를 넣는다면?

이제 Main클래스에서 테스트해봅시다!

객체 선언

객체 선언은 다음과 같이 합니다.

public class Main {
	public static void main(String args[]) {
		Computer c1 = new Computer(); // 객체 선언
	}
}

c1이라는 변수명을 가진 Computer형 객체.
모든 객체는 아래와 같은 형식으로 선언합니다.

  int i = new Int(5); // int i = 5; 와 동일
  String str = new String("EFAS");

그리고 이제 c1의 IP값을 지정해야 합니다.

Computer c1 = new Computer();
c1.ip = "1.234.567.890"; // c1.ip 값 지정

Computer에 있는 ip변수에 접근한 것입니다.

이제 c1의 ip값을 출력문으로 출력해봅시다.

생성자

Computer 클래스의 IP값을 보다 빠르게 초기화 하기 위해 생성자를 이용할 수 있습니다.

Computer 클래스

public class Computer {
	String ip;
	
	public Computer(String ip) { // 생성자!
		this.ip = ip; // this.ip는 전역변수, ip는 매개변수
	}
}

Main 클래스

 public class Main {
	public static void main(String args[]) {
		Computer c1 = new Computer("1.234.567.890");
		System.out.println(c1.ip); // 1.234.567.890
	}
}

Static

static은 어디서나, 초기화 없이도 접근할 수 있는, C언어 포인터보다도 강력한 기능입니다.
보통 접근 빈도가 매우 높은 변수에 사용되거나 중앙 데이터를 관리하는 변수에 사용합니다. 하지만 프로그램이 꺼질때까지 메모리를 할당하고 있기때문에 너무 많은 static 변수는 메모리 누수를 유발합니다.

요약. static은 객체 선언 없이 바로 변수 사용이 가능한 기능!

코드를 고쳐줍시다.

Computer 클래스

public class Computer {
	public static String ip;
}

Main 클래스

 public class Main {
	public static void main(String args[]) {
		Computer.ip = "1.23.247.245";
		System.out.println(Computer.ip); // 1.23.247.245
		Computer.ip = "192.168.0.1";
		System.out.println(Computer.ip); // 192.168.0.1 
	}
}

객체 선언 없이 편하네요!

주의!! 위 코드는 데이터 클래스 용도의 사용으로는 절대 권장하지 않습니다! static은 남발하지 마세요!
하지만 저장용도의 클래스라면 static은 충분히 권장합니다!

클래스 생성

해당 프로젝트 폴더를 열어준 다음, src(우클릭) -> New -> Class

Name칸에 생성할 클래스 이름을 정해주고 Finish버튼을 눌러줍니다.
클래스 이름의 시작은 대문자를 권장합니다
이어서 진행하기

private

public class Computer {
	private String ip; 
}

위 코드는 private가 적용된 코드입니다.

위 코드는 다른 클래스에서 ip값을 조회,수정이 불가능한 코드입니다.
만약 저 ip 접근 지시자를 public으로 바꾼다면 조회,수정 모두 가능합니다.
여기서 살짝 변형해, 조회만 가능한 메소드를 만들어 주는 방법이 있습니다.

  1. public final로 수정 못하게 막아버리기

    public final String ip; // 전역변수만 고쳐주면 됩니다

  2. 조회전용 메소드 만들어주기

     public String getIP() {
 	return ip; // ip는 private인 상태!!
 }

    위 메소드와 ip변수는 같은 클래스이기 때문에 접근 지시자는 의미가 없습니다. 그리고 다른 클래스에서는 private인 ip는 접근하지 못하지만 getIP()에는 접근할 수 있으므로, 조회만 가능한 상태입니다.

주의!! getIP도 값을 받고 나면 수정이 가능합니다. 하지만 getIP의 값은 직통으로 수정이 불가능하며, 설령 변경했다 하더라도 다시 getIP의 값을 받아보면 수정 전 상태입니다.**

예시)

  1. (변수).getIP() = "127.0.0.1" // 에러

  2.  String ip = (변수).getIP(); // "192.168.0.1"
 ip = "172.0.0.2";
 System.out.println((변수).getIP()); // 출력 :  192.168.0.1
